# Sprint Planning

## Overview

The Sprint Planning phase breaks roadmap items into actionable sprint-sized work. AAVA accelerates story writing, capacity planning, and goal setting for each sprint.

This phase covers three core flows:

1. **Story Generation** — AI-assisted user story creation from feature specs
2. **Capacity Planning** — Matching sprint scope to team capacity
3. **Sprint Goal Setting** — Defining clear, measurable sprint objectives

## Inputs

- Roadmap items scheduled for current period
- Team availability and velocity data
- Feature specifications

## Outputs

- Sprint backlog with user stories
- Capacity allocation plan
- Sprint goals tied to OKRs
