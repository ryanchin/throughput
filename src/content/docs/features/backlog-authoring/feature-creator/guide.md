# How to Use the Feature Creator

This guide walks you through generating a feature with decomposed user stories.

## Step 1: Provide Input

- **Title** (required) — Clear name for the feature
- **Description** (required) — What this feature does and the problem it solves

Import context from Ideation, Scope Definition, Goals, PRDs, or parent Epics.

### Examples of Great Input

**Title:** "Automated Health Score Alerts"

**Description:** "When a customer's health score drops below the configured threshold for their tier, automatically notify the assigned CSM via email and Slack with a summary of contributing factors and a suggested next-best-action playbook."

## Step 2: Review Generated Stories

Each user story includes acceptance criteria in Gherkin format, story point estimates, and business value justification.

## Step 3: Refine and Export

- Refine with AI or edit manually
- Export to PDF or push individual stories to Jira

## Tips

- A well-described feature with clear scope produces the best story decomposition
- Review acceptance criteria carefully — they're sprint-ready when generated
