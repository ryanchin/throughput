# Defect Manager

AI-powered defect triage and tracking — classify severity, identify root causes, detect duplicate defects, and track resolution progress from your live Jira or Azure DevOps data.

## Questions It Answers

- "What are our highest-priority defects right now?"
- "Are any of these bugs duplicates?"
- "What's the likely root cause of this defect?"
- "How is our defect resolution trending over time?"

## Example Use Cases

- A QA lead wants to triage a backlog of 50 reported bugs with AI-assisted severity classification and duplicate detection — reducing triage time from hours to minutes.
- An engineering manager wants to spot root cause patterns across defects to identify systemic quality issues.
- A release manager needs a real-time view of open critical and high-severity defects before signing off on a release.

## What Makes It Special

- Pulls defects directly from Jira or Azure DevOps — no manual data entry
- AI classifies severity, suggests root causes, and detects potential duplicates
- Trend analysis shows whether quality is improving or degrading over time
- Integrates with Release Readiness for go/no-go decisions
