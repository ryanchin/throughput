# How to Use the Roadmap Planner

This guide walks you through building a sequenced, capacity-matched product roadmap.

## Step 1: Add Roadmap Items

Pull items from multiple sources:

- **Feasibility Analyses** — Import assessed features with effort estimates
- **Epics and Features** — Pull from the Story Generator
- **Ideation Ideas** — Import top-scoring ideas
- **Custom Entries** — Add items manually with your own estimates

### Examples of Great Configuration

**Custom Entries (when not importing):**
- "SSO Integration — 34 story points, depends on: Identity Provider Research. Assigned to Platform team."
- "Billing Portal v2 — 55 story points, depends on: Payment API Refactor, Design System Update. Assigned to Growth team."
- "Mobile Push Notifications — 21 story points, no dependencies. Assigned to Mobile team."

## Step 2: Configure Team Capacity

Set your planning parameters:

- **Team count** — How many delivery teams are available
- **Velocity** — Story points or effort per sprint per team
- **Sprint length** — Duration of each sprint
- **Buffer percentage** — How much capacity to reserve for unplanned work

## Step 3: Review the Generated Roadmap

Product Studio generates a roadmap with:

- **Sequenced items** — Ordered by dependencies and priority
- **Dependency detection** — Automatic identification of what blocks what
- **Thematic clusters** — Items grouped by related themes
- **Milestone markers** — Key checkpoints in the timeline
- **Capacity matching** — Items fit within team capacity constraints

## Step 4: Adjust and Iterate

Modify the roadmap by changing capacity parameters, reordering items, or adjusting priorities. The system recalculates automatically.

## Tips

- Set a realistic buffer percentage (15-25%) — zero-buffer roadmaps fail in practice
- Import from Feasibility Analysis for the most accurate effort data
- Use thematic clusters to identify natural release groupings
- Review dependency chains carefully — they often determine your critical path
