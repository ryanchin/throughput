# Release Prep

Auto-generates release documentation — release notes, decision logs, and migration guides — from your sprint data and completed work items.

## Questions It Answers

- "What's included in this release?"
- "What do customers need to know about these changes?"
- "What decisions were made during development and why?"
- "Are there migration steps required for this release?"

## Example Use Cases

- A release manager needs customer-facing release notes generated from the sprint's completed Jira tickets — without manually combing through each one.
- A product team wants to auto-generate an internal decision log capturing the trade-offs made during the sprint for future reference.
- An ops team needs migration guides for a release that includes database changes or API breaking changes.

## What Makes It Special

- Pulls completed items directly from Jira or Azure DevOps — no manual data entry
- Generates multiple document types: release notes (customer-facing), decision logs (internal), and migration guides (technical)
- Audience-aware tone — customer release notes read differently than internal decision logs
- Covers what changed, why it changed, and what action is needed
