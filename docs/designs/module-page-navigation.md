# Design: Module → Page Navigation

Generated by /plan-ceo-review on 2026-03-18
Branch: main | Mode: SELECTIVE EXPANSION
Repo: ryanchin/throughput

## Problem

Each lesson (module) is a single scrollable wall of text — 2000-4000 words with no sense of progress. Learners disengage. The training experience should feel like clicking through bite-sized pages, not reading a blog post.

## Solution

**Client-side page splitting.** Split lesson content at `## heading` boundaries during render. Each `## section` becomes its own page with Previous/Next navigation, a progress bar, and fade transitions. Zero DB changes — works with all existing content immediately.

## Scope

### Core
- `splitContentIntoPages()` — splits Tiptap JSON at `## heading` nodes
- `SectionPaginator` — Previous/Next, progress bar, page title, keyboard nav, fade transitions
- Update lesson page to render one page at a time
- Quiz integrated as final page in the paginator

### Accepted Expansions
1. Progress bar within module (thin cyan bar)
2. Keyboard navigation (Left/Right arrow keys)
3. AI prompt updates for 5-10 sections of 400-800 words each
4. Section mini-TOC in sidebar (expandable page list under each module)
5. Quiz as dedicated final page in paginator
6. Smooth 150ms fade-in transitions

## Architecture

```
NO DB CHANGES — rendering change only

Existing:                          New:
lessons.content (Tiptap JSON)      splitContentIntoPages(content)
         │                                    │
         ▼                                    ▼
  LessonViewer (full)              pages: TiptapJSON[] (one per ## section)
                                              │
                                              ▼
                                   SectionPaginator
                                   ├── Progress bar (2px, ambient)
                                   ├── Page title (from heading)
                                   ├── LessonViewer (single page)
                                   ├── Pagination bar (Prev/Next)
                                   ├── Keyboard nav (←/→)
                                   └── Fade transition (150ms)
```

### Interaction States

| Feature | State | What User Sees |
|---------|-------|---------------|
| Progress bar | Always | 2px cyan fill on muted track, width = (page/total)*100% |
| Previous (page 1) | Hidden | Not rendered (no disabled button — just absent) |
| Previous (page 2+) | Active | Secondary button: `border-border bg-background hover:border-accent/30` |
| Next (middle pages) | Active | Primary button: `bg-accent text-background hover:bg-accent-hover` |
| Next (last content, has quiz) | Active | Text: "Take Quiz →" with quiz icon. Same primary style. |
| Next (last page, no quiz) | Active | Text: "Complete Module". Same primary style. |
| Single-page module | Hidden | No paginator, no progress bar — renders like old full-content view |
| Page transition | 150ms | Opacity 0→1 fade. `prefers-reduced-motion: instant swap` |
| Keyboard nav | Ambient | Left/Right arrows. Only fires when focus is on body (not input/textarea) |

### User Journey

| Step | User Does | User Feels | Design Supports With |
|------|-----------|-----------|---------------------|
| 1 | Opens module | Oriented | First page is short, progress bar shows 1 of N |
| 2 | Reads first page | Manageable | Content is one screen, not a wall |
| 3 | Clicks Next | Accomplishment | Progress bar advances, smooth fade, new title |
| 4 | Scans sidebar | In control | All pages visible, current highlighted cyan |
| 5 | Reaches last content page | Anticipation | "Take Quiz →" signals transition |
| 6 | Takes quiz | Focused | Quiz renders in same content area |
| 7 | Passes quiz | Pride | Success state, module complete |
| 8 | Module complete | Accomplished | Checkmark in sidebar, next module expands |

### Responsive Behavior

| Viewport | Sidebar | Paginator | Progress Bar |
|----------|---------|-----------|-------------|
| Desktop (≥1024px) | Visible, shows page TOC | Prev left, count center, Next right | 2px top of content |
| Tablet (768-1023px) | Collapsible hamburger, page TOC when open | Same layout, slightly smaller buttons | 2px top of content |
| Mobile (<768px) | Hidden, sheet overlay on tap | Side-by-side buttons (44px min-h), count above | h-1 for touch visibility |

### Accessibility

- Paginator: `role="navigation"` + `aria-label="Lesson page navigation"`
- Previous/Next: `aria-label="Previous page"` / `aria-label="Next page: [title]"`
- Progress bar: `role="progressbar"` + `aria-valuenow` + `aria-valuemin=1` + `aria-valuemax`
- Keyboard: Left/Right arrows, only when `activeElement` is body/content (not inputs)
- Page change: `aria-live="polite"` region announces new page title
- Sidebar: `role="list"` + `aria-current="page"` on current item
- Reduced motion: `prefers-reduced-motion` → instant swap, no fade
- All buttons: 44px minimum touch target on mobile

## Key Decisions

| Decision | Choice | Reasoning |
|----------|--------|-----------|
| Approach | Client-side splitting | Zero DB changes, works with existing content |
| Split boundary | `## heading` nodes | Natural content structure, AI generates these |
| URL strategy | `?page=3` query param | Preserves lesson URL, bookmarkable |
| Quiz integration | Final page in paginator | Seamless flow |
| Fallback | No headings = single page, no paginator | Clean, honest — no "Page 1 of 1" |
| Keyboard nav | Left/Right arrows | Don't fire in inputs/textareas |
| Progress tracking | Per-lesson (not per-page) | Keeps existing system |
| Previous on page 1 | Hidden (not disabled) | Subtraction default |
| Sidebar TOC | Only active module expanded | Keeps sidebar manageable |
| Progress bar style | 2px line (not thick bar) | Ambient, not distracting |
