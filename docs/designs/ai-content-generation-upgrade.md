## Problem

The AI generation experience is half-built. Courses have a basic toggle (title + description only — no custom instructions, no file context, no course awareness). Certifications have zero AI support. Admins can't upload reference docs or tell the AI what tone/style to use. This is the platform's core differentiator and it needs to be complete.

## Solution

A **Unified AI Context Panel** — a shared component used by CourseForm, CertificationForm, and LessonEditor that provides:
1. **Style presets** — quick-select buttons (Technical, Conversational, Assessment-focused, Beginner-friendly)
2. **Instructions textarea** — free-form guidance to the LLM
3. **File upload** — PDF/DOCX reference materials with extracted text preview and word count
4. **Course picker** — multi-select with search, lesson count, and content preview
5. **Smart context management** — auto-summarizes if combined context exceeds token limits

## Scope

### Core Deliverables
- Shared `AiContextPanel` component
- Extend `/api/admin/generate/course` with instructions, file context, course context
- New `/api/admin/generate/certification` for question pool generation
- File text extraction utility (PDF/DOCX → plain text)
- Update CourseForm to use AiContextPanel
- Add AI toggle + AiContextPanel to certification create/edit

### Accepted Expansions
1. **AI lesson regeneration with context** — upgrade `/api/admin/generate/lesson` and lesson editor to use shared context
2. **Generate questions for existing cert tracks** — "Generate Questions" button on cert track editor
3. **Smart course picker** — searchable list with title, lesson count, content preview
4. **AI tone/style presets** — row of preset buttons above instructions textarea
5. **File content preview** — extracted text + word count shown after upload
6. **Generation history / audit trail** — `generation_logs` table + admin dashboard tab

### Generation Overlay — Progressive Status

```

```

### User Journey

| Step | User Does | User Feels | Design Supports With |
|------|-----------|-----------|---------------------|
| 1 | Sees "Generate with AI" toggle | Curious, hopeful | Prominent toggle, cyan accent, value subtitle |
| 2 | Toggles AI ON | Engaged | Panel expands smoothly (200ms ease-out) |
| 3 | Clicks style preset | "This is easy" | Instant visual feedback, progress feeling |
| 4 | Types instructions | In control | Guiding placeholder, no restrictive limits |
| 5 | Uploads file | Smart, leveraged | Progress → preview proves AI understood it |
| 6 | Selects courses | Thorough | Word counts show substance of context |
| 7 | Clicks "Generate" | Anticipation | Progressive status, context summary shown |
| 8 | Sees draft in editor | Impressed | Content quality validates their inputs |

### Accessibility

- Style presets: `role="radiogroup"` with `role="radio"`, arrow key navigation
- File upload: `aria-label="Upload reference document"`, Enter/Space to open picker
- Course checkboxes: shadcn `Checkbox` with `aria-label`, keyboard-navigable list
- File preview: `aria-live="polite"` for extracted text announcement
- Generation overlay: `aria-live="assertive"` for status, `role="status"`
- All touch targets: 44px minimum on mobile
- Color contrast: WCAG AA (satisfied by dark theme tokens)

## Key Decisions

| Decision | Choice | Reasoning |
|----------|--------|-----------|
| Shared vs separate components | Shared AiContextPanel | DRY, consistent UX |
| Token overflow strategy | Smart truncation with LLM summary | Better output quality |
| File storage | Temp processing only, no persistent storage | Files extracted inline, discarded |
| Generation logging | Separate table, non-blocking writes | Audit trail shouldn't block generation |
| File preview length | 3-line preview + "Show more" expand | Builds trust without clutter |
| Generation history location | Tab on admin dashboard | Discoverable without nav bloat |
| Panel animation | 200ms ease-out expand/collapse | Smooth, not jarring |

## Database Changes
- New `generation_logs` table (id, admin_id, generation_type, inputs, output_summary, model, tokens_used, duration_ms, created_at)
- No changes to existing tables

## Error Handling
- File parse failures → 400 with clear message ("Could not read this file")
- LLM timeout on summary → degrade to simple truncation (transparent to admin)
- LLM invalid response → 500 with "Generation failed. Try again." banner (keep form state)
- Generation log write failure → log error, don't block generation
- Generation failure → preserve all admin inputs, show error with [Try again] and [Edit inputs]
